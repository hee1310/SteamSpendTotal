每次想统计自己在Steam上一共花了多少钱都十分麻烦：官方账单分页展示，需要手动不停点加载更多，再逐条区分消费、退款，手动计算金额。
为此我写了一款轻量油猴脚本 **Steam消费总额查看**，自动拉取全部账单、区分订单类型、自动计算总花费与退款，支持多区服货币、多语言、本地缓存与多重防卡死机制。

## 脚本核心功能
### 一键全量加载账单统计✨
进入Steam [消费记录页面](https://store.steampowered.com/account/history) 后页面会自动渲染「查看消费统计」按钮，点击自动执行完整统计流程：
- 循环自动点击「加载更多」分页按钮拉取全部历史账单，内置120页最大限制，避免无限加载导致页面卡死
- 自动识别页面货币符号，适配全球各区服不同货币
- 智能区分消费订单与退款订单，自动算出三项核心金额：
- 总支出：所有下单原始金额（包含后续退款的订单）
- 累计退款总额：所有退款订单金额合计
- 实际净支出 = 总支出 - 累计退款（真实消费金额）

### 本地缓存加速机制⚡
- 统计结果本地缓存5分钟，短时间重复点击不会重新加载分页，大幅提升使用速度，减少页面卡顿
- 缓存过期后再次点击，会重新拉取完整账单并重新计算最新数据

### 多语言自动适配🌍
脚本自动读取Steam页面语言，弹窗、按钮文字自动切换，目前支持三种语言：
- 简体中文（zh-CN）
- 繁体中文（zh-TW）
- English

### 多重防卡死容错设计🛡️
- 每页加载间隔450ms，给Steam页面渲染数据预留缓冲时间，防止请求过快被限制
- MutationObserver节点监听 + 800ms定时DOM巡检双重保障按钮正常渲染显示
- 30秒后自动关闭定时巡检，降低浏览器性能占用
- 全局捕获页面加载异常，弹出错误提示引导刷新页面重试
- 加载锁拦截重复点击，避免并发重复请求账单数据

## 自定义参数配置
可自行修改代码顶部 `cfg` 对象调整运行参数。

```javascript
const cfg = {
    maxPage: 120,    // 最大加载账单页数，账单极多可上调，建议不超200
    wait: 450,       // 每页加载等待毫秒，网络差可增大至800~1000
    cacheTime: 300000, // 统计缓存有效期ms，300000=5分钟
    domTick: 800     // DOM定时检测间隔，一般无需修改
};
```

# 安装与使用教程
## 前置条件
浏览器安装 **Tampermonkey（油猴）** 扩展，Chrome、Edge、Firefox、Opera 均支持。

## 一键安装脚本
Greasy Fork 脚本地址：<https://greasyfork.org/scripts/583431>
打开链接点击「安装此脚本」即可一键导入油猴。

## 使用步骤
- 登录 Steam 网页端
- 打开消费记录页面：<https://store.steampowered.com/account/history>
- 页面下方会出现「查看消费统计」按钮
- 点击按钮等待自动加载全部账单
- 加载完成弹窗展示总支出、退款、实际净花费


# 写在最后
脚本完全本地运行，不会上传任何账单、账号数据至第三方，无广告、无后台请求，轻量化无侵入页面。
