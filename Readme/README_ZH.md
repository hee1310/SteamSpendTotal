每次想要統計自己在Steam上一共花了多少錢都十分麻煩：官方帳單採用分頁展示，需要手動不停點擊載入更多，再逐筆區分消費、退款，手動計算金額。
為此我撰寫了一款輕量油猴腳本 **Steam消費總額查看**，自動拉取全部帳單、區分訂單類型、自動計算總花費與退款，支援多區服貨幣、多語言、本地快取與多重防卡死機制。

## 腳本核心功能
### 一鍵全量載入帳單統計✨
進入Steam消費紀錄頁面（https://store.steampowered.com/account/history）後頁面會自動渲染「查看消費統計」按鈕，點擊自動執行完整統計流程：
- 循環自動點擊「載入更多」分頁按鈕拉取全部歷史帳單，內建120頁最大限制，避免無限載入導致頁面卡死
- 自動識別頁面貨幣符號，適配全球各地區不同貨幣
- 智慧區分消費訂單與退款訂單，自動算出三項核心金額：
	- 總支出：所有下單原始金額（包含後續退款的訂單）
	- 累計退款總額：所有退款訂單金額合計
	- 實際淨支出 = 總支出 - 累計退款（真實消費金額）

### 本地快取加速機制⚡
- 統計結果本地快取5分鐘，短時間重複點擊不會重新載入分頁，大幅提升使用速度，減少頁面卡頓
- 快取過期後再次點擊，會重新拉取完整帳單並重新計算最新數據

### 多語言自動適配🌍
腳本自動讀取Steam頁面語言，彈窗、按鈕文字自動切換，目前支援三種語言：
- 簡體中文（zh-CN）
- 繁體中文（zh-TW）
- English

### 多重防卡死容錯設計🛡️
- 每頁載入間隔450ms，給予Steam頁面渲染資料緩衝時間，避免請求過快被限制
- MutationObserver節點監聽 + 800ms定時DOM巡檢雙重保障按鈕正常渲染顯示
- 30秒後自動關閉定時巡檢，降低瀏覽器效能占用
- 全域捕獲頁面載入異常，彈出錯誤提示引導重整頁面重試
- 載入鎖攔截重複點擊，避免並發重複請求帳單數據

## 自訂參數設定
可自行修改程式碼頂部 `cfg` 物件調整執行參數。

```javascript
const cfg = {
    maxPage: 120,    // 帳單讀取最大頁數，帳單資料過多可調高數值，上限200
    wait: 450,       // 單頁載入等候時間，網路延遲不穩時調整至800~1000
    cacheTime: 300000, // 快取資料有效時長(毫秒)，300000等同五分鐘
    domTick: 800     // 頁面元件輪詢檢測間隔，一般狀況無需調整
};
```

# 安裝與使用教學
## 前置條件
瀏覽器安裝 **Tampermonkey（油猴）** 擴充，Chrome、Edge、Firefox、Opera 皆支援。

## 一鍵安裝腳本
Greasy Fork 腳本網址：<https://greasyfork.org/scripts/583431>
打開連結點擊「安裝此腳本」即可一鍵匯入油猴。

## 使用步驟
- 登入 Steam 網頁端
- 打開消費紀錄頁面：<https://store.steampowered.com/account/history>
- 頁面下方會出現「查看消費統計」按鈕
- 點擊按鈕等待自動載入全部帳單
- 載入完成彈窗展示總支出、退款、實際淨花費

# 寫在最後
腳本完全本地執行，不會上傳任何帳單、帳號數據至第三方，無廣告、無後台請求，輕量化不侵入頁面。
如果你也想快速統計自己 Steam 多年遊戲消費，不妨安裝試試，有使用問題可以在[issues](https://github.com/hee1310/SteamSpendTotal/issues)提出。
